CRG × Finergi
Token Advance Integration

Maquettes UI mobiles pour l'intégration du flux d'avance de tokens d'électricité, incluant les flux d'achat à crédit (Token Advance) et de remboursement (Repayment). Chaque écran est annoté avec l'endpoint Finergi à appeler.

Environmentapi-dev.finergi.cloud
Auth methodOAuth2 / Bearer JWT
CurrencyGNF (Guinean Franc)
Target platformsiOS · Android
— Architecture —

L'app mobile parle uniquement à Finergi

Pour ce nouveau parcours "Token Advance", l'app mobile fait un seul appel API — vers Finergi. Finergi orchestre tout en backend : génération du token via EDG (B2B) et gestion du Mobile Money. Le parcours existant "Recharge directe" (achat direct via EDG) reste inchangé.

📱 App mobile CRG
Parle uniquement à Finergi
UI + appels API Finergi. Pas d'appel direct à EDG pour ce parcours.
🏢 Finergi (orchestrateur)
Gère le prêt + appelle EDG
Crédit, dette, remboursement. Appelle EDG en backend pour générer le token.
⚡ EDG (génère le token)
Produit le token cryptographique
Invisible pour l'app. Le token reçu via Finergi est un vrai token EDG.
⚠ Important
Le token retourné par Finergi est un vrai token EDG (généré par EDG, transité par Finergi). Il a exactement le même format que les tokens du parcours "Recharge directe" existant dans Credit Money — réutiliser le même composant UI d'affichage.
